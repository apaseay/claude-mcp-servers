# personal-tools MCP Server

A personal Model Context Protocol (MCP) server built with TypeScript, connecting Claude Desktop to real-world tools via free public APIs and Node.js built-in modules. No work credentials required.

This project demonstrates the full MCP server pattern — from project setup and TypeScript compilation through to Claude Desktop integration and live tool usage. Every additional MCP server (Okta, Jira, SentinelOne) follows the exact same structure.

---

## Tools

| Tool | Description | Source |
|------|-------------|--------|
| `get_weather` | Current weather for any city — temperature, conditions, humidity, wind | [wttr.in](https://wttr.in) (free, no key) |
| `get_github_repos` | List public repositories for any GitHub username | GitHub public API (no key) |
| `get_news_headlines` | Latest headlines from RSS feeds — BBC, Hacker News, TechCrunch, Guardian, RTE or any custom URL | RSS parsing (no key) |
| `get_okta_user` | Look up an Okta user by email — name, status, last login | Okta API (requires credentials) |
| `check_okta_mfa` | Check whether an Okta user has MFA enrolled and which factors are registered | Okta API (requires credentials) |
| `list_okta_inactive_users` | Find active Okta users who have not logged in for N days | Okta API (requires credentials) |
| `lookup_ip` | Geolocation and ISP information for any public IP address | [ip-api.com](http://ip-api.com) (free, no key) |
| `dns_lookup` | A, MX, and TXT records for any domain | Node.js built-in `dns` module |
| `generate_password` | Cryptographically secure random password — configurable length and symbol inclusion | Node.js built-in `crypto` module |

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/apaseay/personal-tools.git
cd personal-tools
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```bash
touch .env
```

Add your Okta credentials (only required for Okta tools — all other tools work without this):

```env
OKTA_ORG_URL=https://your-org.okta.com
OKTA_API_TOKEN=your_api_token_here
```

Leave this file empty or omit it entirely if you are not using the Okta tools.

### 4. Build the server

```bash
npm run build
```

This compiles `src/index.ts` to `dist/index.js`.

### 5. Verify the build

```bash
node dist/index.js
```

You should see:

```
personal-tools MCP server running (v3 — full suite)
```

Press `Ctrl + C` to stop.

## Claude Desktop Integration

Open your Claude Desktop config file:

```bash
open ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

Add the following inside the `mcpServers` object (replace `/Users/yourname` with your actual home directory):

```json
{
  "mcpServers": {
    "personal-tools": {
      "command": "node",
      "args": ["/Users/yourname/projects/claude-mcp-servers/personal-tools/dist/index.js"]
    }
  }
}
```

Restart Claude Desktop. The tools will appear automatically in your next conversation.

## Development

### Rebuild after changes

```bash
npm run build
```

Always rebuild and restart Claude Desktop after editing `src/index.ts`.

### Adding a new tool

1. Add the tool definition to the `ListToolsRequestSchema` handler in `src/index.ts`
2. Add the corresponding `case` block to the `CallToolRequestSchema` handler
3. Run `npm run build`
4. Restart Claude Desktop

### Important: stdout discipline

MCP servers communicate with Claude via JSON-RPC over stdout. Never use `console.log()` in your server code — it will break the connection. Use `console.error()` for all debug and status output, which writes to stderr and is ignored by the MCP client.

## Notes

- The Okta tools use basic list and filter endpoints and require a Read Only Admin token or equivalent
- `lookup_ip` uses ip-api.com which allows 45 requests per minute on the free tier
- `dns_lookup` uses Node.js native DNS resolution — no external API or rate limits apply
- `generate_password` uses Node.js `crypto.randomBytes` for cryptographically secure generation
