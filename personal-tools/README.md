# personal-tools MCP Server

A personal Model Context Protocol (MCP) server built with TypeScript, connecting Claude Desktop to real-world tools via free public APIs and Node.js built-ins. No work credentials required for the core tools.

This project demonstrates the full MCP server pattern — from project setup and TypeScript compilation through to Claude Desktop integration and live tool usage. Every additional MCP server (Okta, Jira, SentinelOne) follows the exact same structure.

> **What you need:** Just your Mac, Node.js 18+, and Claude Desktop. No accounts, no tokens, no sign-ups for the core tools.

---

## What This Server Provides

| Tool | Description | Source |
|------|-------------|--------|
| `get_weather` | Current weather for any city — temperature, conditions, humidity, wind | [wttr.in](https://wttr.in) (free, no key) |
| `get_github_repos` | List public repositories for any GitHub username | GitHub public API (no key) |
| `get_news_headlines` | Latest headlines from BBC, Hacker News, TechCrunch, Guardian, RTE or any custom RSS URL | RSS parsing (no key) |
| `lookup_ip` | Geolocation and ISP information for any public IP address | [ip-api.com](http://ip-api.com) (free, no key) |
| `dns_lookup` | A, MX, and TXT records for any domain | Node.js built-in `dns` module |
| `generate_password` | Cryptographically secure random password — configurable length and symbol inclusion | Node.js built-in `crypto` module |
| `get_okta_user` | Look up an Okta user by email — name, status, last login | Okta API (requires credentials) |
| `check_okta_mfa` | Check whether an Okta user has MFA enrolled and which factors are registered | Okta API (requires credentials) |
| `list_okta_inactive_users` | Find active Okta users who have not logged in for N days | Okta API (requires credentials) |

## Phase 2 — Create the Server File

### Step 7 — Create src/index.ts


Open the file and paste in the complete server code below. Every section includes comments explaining what it does.

```typescript
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from '@modelcontextprotocol/sdk/types.js';
import RSSParser from 'rss-parser';
import * as dotenv from 'dotenv';
import { fileURLToPath } from 'url';
import { dirname, join } from 'path';
import * as dns from 'dns';
import * as crypto from 'crypto';
import { promisify } from 'util';

// ─── Load .env ────────────────────────────────────────────────────────────────
const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);
dotenv.config({ path: join(__dirname, '..', '.env') });

// ─── Promisify DNS methods ─────────────────────────────────────────────────────
const dnsResolve4 = promisify(dns.resolve4);
const dnsResolveMx = promisify(dns.resolveMx);
const dnsResolveTxt = promisify(dns.resolveTxt);

// ─── RSS Parser instance ───────────────────────────────────────────────────────
const rssParser = new RSSParser();

// ─── Okta credentials ─────────────────────────────────────────────────────────
const OKTA_ORG_URL = process.env.OKTA_ORG_URL;
const OKTA_API_TOKEN = process.env.OKTA_API_TOKEN;

// ─── Okta helper: authenticated GET ───────────────────────────────────────────
async function oktaGet(endpoint: string): Promise<any> {
  if (!OKTA_ORG_URL || !OKTA_API_TOKEN) {
    throw new Error('Okta credentials not configured. Add OKTA_ORG_URL and OKTA_API_TOKEN to your .env file.');
  }
  const response = await fetch(`${OKTA_ORG_URL}/api/v1${endpoint}`, {
    headers: {
      'Authorization': `SSWS ${OKTA_API_TOKEN}`,
      'Accept': 'application/json',
    },
  });
  if (!response.ok) {
    throw new Error(`Okta API error ${response.status}: ${await response.text()}`);
  }
  return response.json();
}

// ─── Okta helper: format a user object ────────────────────────────────────────
function formatUser(user: any): string {
  const p = user.profile;
  const lastLogin = user.lastLogin ? new Date(user.lastLogin).toLocaleDateString('en-IE') : 'Never';
  return [
    `Name:       ${p.firstName} ${p.lastName}`,
    `Email:      ${p.email}`,
    `Status:     ${user.status}`,
    `Last Login: ${lastLogin}`,
    `Created:    ${new Date(user.created).toLocaleDateString('en-IE')}`,
  ].join('\n');
}

// ─── Create the MCP server ─────────────────────────────────────────────────────
const server = new Server(
  { name: 'personal-tools', version: '3.0.0' },
  { capabilities: { tools: {} } }
);

// ─── TOOL DEFINITIONS ──────────────────────────────────────────────────────────
// This is the "menu" — it tells Claude what tools exist and what each one does.
// Claude reads these descriptions to decide when to call each tool.

server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: 'get_weather',
      description:
        'Get the current weather for any city in the world. ' +
        'Returns temperature, conditions, humidity, and wind speed.',
      inputSchema: {
        type: 'object',
        properties: {
          city: { type: 'string', description: 'City name, e.g. Dublin, Lagos, London' },
        },
        required: ['city'],
      },
    },
    {
      name: 'get_github_repos',
      description:
        'List the public GitHub repositories for any GitHub username. ' +
        'Shows repo names, descriptions, stars, and last updated date.',
      inputSchema: {
        type: 'object',
        properties: {
          username: { type: 'string', description: 'GitHub username, e.g. apaseay' },
          limit: { type: 'number', description: 'Number of repos to return (default: 10, max: 30)' },
        },
        required: ['username'],
      },
    },
    {
      name: 'get_news_headlines',
      description:
        'Fetch the latest news headlines from an RSS feed. ' +
        'Built-in shortcuts: bbc, hn, techcrunch, guardian, rte. ' +
        'Or provide a full RSS URL directly.',
      inputSchema: {
        type: 'object',
        properties: {
          source: {
            type: 'string',
            description: 'Shortcut name (bbc, hn, techcrunch, guardian, rte) or a full RSS URL',
          },
          limit: { type: 'number', description: 'Number of headlines to return (default: 10)' },
        },
        required: ['source'],
      },
    },
    {
      name: 'lookup_ip',
      description: 'Get geolocation and ISP information for any public IP address.',
      inputSchema: {
        type: 'object',
        properties: {
          ip: { type: 'string', description: 'Public IP address, e.g. 8.8.8.8' },
        },
        required: ['ip'],
      },
    },
    {
      name: 'dns_lookup',
      description: 'Look up DNS records for a domain — A, MX, and TXT records.',
      inputSchema: {
        type: 'object',
        properties: {
          domain: { type: 'string', description: 'Domain name, e.g. github.com' },
        },
        required: ['domain'],
      },
    },
    {
      name: 'generate_password',
      description: 'Generate a cryptographically secure random password.',
      inputSchema: {
        type: 'object',
        properties: {
          length: { type: 'number', description: 'Password length (default: 16)' },
          includeSymbols: { type: 'boolean', description: 'Include symbols (default: true)' },
        },
      },
    },
    {
      name: 'get_okta_user',
      description: 'Look up an Okta user by email. Returns name, status, and last login. Requires Okta credentials in .env.',
      inputSchema: {
        type: 'object',
        properties: {
          email: { type: 'string', description: "User's email address" },
        },
        required: ['email'],
      },
    },
    {
      name: 'check_okta_mfa',
      description: 'Check whether an Okta user has MFA enrolled and which factors are registered. Requires Okta credentials in .env.',
      inputSchema: {
        type: 'object',
        properties: {
          email: { type: 'string', description: "User's email address" },
        },
        required: ['email'],
      },
    },
    {
      name: 'list_okta_inactive_users',
      description: 'Find active Okta users who have not logged in for a specified number of days. Requires Okta credentials in .env.',
      inputSchema: {
        type: 'object',
        properties: {
          days: { type: 'number', description: 'Days of inactivity, e.g. 30' },
          limit: { type: 'number', description: 'Max users to return (default: 20)' },
        },
        required: ['days'],
      },
    },
  ],
}));

// ─── TOOL HANDLERS ─────────────────────────────────────────────────────────────
// When Claude calls a tool, this code runs.

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  try {
    const args = request.params.arguments as Record<string, any>;

    switch (request.params.name) {

      // ── get_weather ──────────────────────────────────────────────────────────
      // Uses wttr.in — a free weather API that needs no API key.
      case 'get_weather': {
        const city = args.city as string;
        const response = await fetch(
          `https://wttr.in/${encodeURIComponent(city)}?format=j1`,
          { headers: { 'User-Agent': 'personal-tools-mcp/1.0' } }
        );
        if (!response.ok) throw new Error(`Weather API error for: ${city}`);
        const data = await response.json() as any;
        const current = data.current_condition?.[0];
        const area = data.nearest_area?.[0];
        if (!current) return { content: [{ type: 'text', text: `No weather data for: ${city}` }] };
        const locationName = area
          ? `${area.areaName?.[0]?.value}, ${area.country?.[0]?.value}`
          : city;
        return {
          content: [{ type: 'text', text: [
            `Weather for ${locationName}`,
            `─────────────────────────`,
            `Condition:    ${current.weatherDesc?.[0]?.value}`,
            `Temperature:  ${current.temp_C}°C (feels like ${current.FeelsLikeC}°C)`,
            `Humidity:     ${current.humidity}%`,
            `Wind:         ${current.windspeedKmph} km/h ${current.winddir16Point}`,
            `Visibility:   ${current.visibility} km`,
            `Cloud cover:  ${current.cloudcover}%`,
          ].join('\n') }],
        };
      }

      // ── get_github_repos ─────────────────────────────────────────────────────
      // Uses the GitHub public API — no authentication needed for public repos.
      case 'get_github_repos': {
        const username = args.username as string;
        const limit = Math.min(args.limit ?? 10, 30);
        const response = await fetch(
          `https://api.github.com/users/${encodeURIComponent(username)}/repos?sort=updated&per_page=${limit}`,
          { headers: { 'Accept': 'application/vnd.github.v3+json', 'User-Agent': 'personal-tools-mcp/1.0' } }
        );
        if (response.status === 404) return { content: [{ type: 'text', text: `GitHub user not found: ${username}` }] };
        if (!response.ok) throw new Error(`GitHub API error: ${response.status}`);
        const repos = await response.json() as any[];
        if (!repos.length) return { content: [{ type: 'text', text: `No public repos for: ${username}` }] };
        const lines = repos.map((r: any) => {
          const updated = new Date(r.updated_at).toLocaleDateString('en-IE');
          const stars = r.stargazers_count > 0 ? ` ⭐ ${r.stargazers_count}` : '';
          const desc = r.description ? `\n     ${r.description}` : '';
          return `• ${r.name}${stars} — updated ${updated}${desc}`;
        });
        return { content: [{ type: 'text', text: [`GitHub repos for @${username}:`, '─'.repeat(40), ...lines].join('\n') }] };
      }

      // ── get_news_headlines ───────────────────────────────────────────────────
      // Uses rss-parser to read any public RSS feed.
      case 'get_news_headlines': {
        const source = args.source as string;
        const limit = args.limit ?? 10;
        const RSS_SOURCES: Record<string, string> = {
          bbc:        'https://feeds.bbci.co.uk/news/rss.xml',
          hn:         'https://hnrss.org/frontpage',
          techcrunch: 'https://techcrunch.com/feed/',
          guardian:   'https://www.theguardian.com/world/rss',
          rte:        'https://www.rte.ie/rss/news.xml',
        };
        const feedUrl = RSS_SOURCES[source.toLowerCase()] ?? source;
        let feed: any;
        try {
          feed = await rssParser.parseURL(feedUrl);
        } catch {
          throw new Error(`Could not load RSS feed: ${feedUrl}. Available shortcuts: ${Object.keys(RSS_SOURCES).join(', ')}`);
        }
        const items = feed.items.slice(0, limit);
        const lines = items.map((item: any, i: number) => {
          const date = item.pubDate ? ` [${new Date(item.pubDate).toLocaleDateString('en-IE')}]` : '';
          return `${i + 1}. ${item.title}${date}`;
        });
        return { content: [{ type: 'text', text: [`Headlines from ${feed.title ?? feedUrl}:`, '─'.repeat(40), ...lines].join('\n') }] };
      }

      // ── lookup_ip ────────────────────────────────────────────────────────────
      // Uses ip-api.com — free, no key, 45 requests/minute limit.
      case 'lookup_ip': {
        const ip = args.ip as string;
        const response = await fetch(`http://ip-api.com/json/${encodeURIComponent(ip)}`);
        if (!response.ok) throw new Error(`IP lookup failed for: ${ip}`);
        const data = await response.json() as any;
        if (data.status === 'fail') return { content: [{ type: 'text', text: `Could not look up IP: ${ip}. ${data.message ?? ''}` }] };
        return {
          content: [{ type: 'text', text: [
            `IP Information for ${ip}`,
            `─────────────────────────`,
            `Location:   ${data.city}, ${data.regionName}, ${data.country}`,
            `ISP:        ${data.isp}`,
            `Org:        ${data.org}`,
            `Timezone:   ${data.timezone}`,
            `Lat/Lon:    ${data.lat}, ${data.lon}`,
          ].join('\n') }],
        };
      }

      // ── dns_lookup ───────────────────────────────────────────────────────────
      // Uses Node.js native DNS — no external API or rate limits.
      case 'dns_lookup': {
        const domain = args.domain as string;
        const results: string[] = [`DNS records for ${domain}`, '─'.repeat(40)];
        try {
          const aRecords = await dnsResolve4(domain);
          results.push(`A Records:\n${aRecords.map(r => `  ${r}`).join('\n')}`);
        } catch { results.push('A Records: none'); }
        try {
          const mxRecords = await dnsResolveMx(domain);
          results.push(`MX Records:\n${mxRecords.map(r => `  ${r.exchange} (priority ${r.priority})`).join('\n')}`);
        } catch { results.push('MX Records: none'); }
        try {
          const txtRecords = await dnsResolveTxt(domain);
          results.push(`TXT Records:\n${txtRecords.map(r => `  ${r.join('')}`).join('\n')}`);
        } catch { results.push('TXT Records: none'); }
        return { content: [{ type: 'text', text: results.join('\n') }] };
      }

      // ── generate_password ────────────────────────────────────────────────────
      // Uses Node.js crypto.randomBytes — cryptographically secure.
      case 'generate_password': {
        const length = args.length ?? 16;
        const includeSymbols = args.includeSymbols !== false;
        const chars = 'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789' +
          (includeSymbols ? '!@#$%^&*()-_=+[]{}|;:,.<>?' : '');
        const bytes = crypto.randomBytes(length);
        const password = Array.from(bytes).map(b => chars[b % chars.length]).join('');
        return {
          content: [{ type: 'text', text: [
            `Generated Password:`,
            `─────────────────────────`,
            password,
            ``,
            `Length:  ${length} characters`,
            `Symbols: ${includeSymbols ? 'included' : 'excluded'}`,
          ].join('\n') }],
        };
      }

      // ── get_okta_user ────────────────────────────────────────────────────────
      case 'get_okta_user': {
        const email = args.email as string;
        const users = await oktaGet(`/users?filter=profile.login+eq+"${encodeURIComponent(email)}"`);
        if (!users?.length) return { content: [{ type: 'text', text: `No Okta user found: ${email}` }] };
        return { content: [{ type: 'text', text: formatUser(users[0]) }] };
      }

      // ── check_okta_mfa ───────────────────────────────────────────────────────
      case 'check_okta_mfa': {
        const email = args.email as string;
        const users = await oktaGet(`/users?filter=profile.login+eq+"${encodeURIComponent(email)}"`);
        if (!users?.length) return { content: [{ type: 'text', text: `No Okta user found: ${email}` }] };
        const user = users[0];
        const name = `${user.profile.firstName} ${user.profile.lastName}`;
        const factors = await oktaGet(`/users/${user.id}/factors`);
        if (!factors?.length) return { content: [{ type: 'text', text: `${name} (${email}) has NO MFA factors enrolled.` }] };
        const list = factors.map((f: any) => `  - ${f.factorType} via ${f.provider} — Status: ${f.status}`).join('\n');
        return { content: [{ type: 'text', text: `${name} has ${factors.length} MFA factor(s):\n${list}` }] };
      }

      // ── list_okta_inactive_users ─────────────────────────────────────────────
      case 'list_okta_inactive_users': {
        const days = args.days as number;
        const limit = Math.min(args.limit ?? 20, 50);
        const cutoff = new Date();
        cutoff.setDate(cutoff.getDate() - days);
        const users = await oktaGet(`/users?filter=status+eq+"ACTIVE"&limit=200`);
        if (!users?.length) return { content: [{ type: 'text', text: 'No active users found.' }] };
        const inactive = users
          .filter((u: any) => !u.lastLogin || new Date(u.lastLogin) < cutoff)
          .sort((a: any, b: any) =>
            (a.lastLogin ? new Date(a.lastLogin).getTime() : 0) -
            (b.lastLogin ? new Date(b.lastLogin).getTime() : 0)
          )
          .slice(0, limit);
        if (!inactive.length) return { content: [{ type: 'text', text: `No users inactive for ${days}+ days.` }] };
        const lines = inactive.map((u: any) => {
          const lastLogin = u.lastLogin ? new Date(u.lastLogin).toLocaleDateString('en-IE') : 'Never';
          return `• ${u.profile.firstName} ${u.profile.lastName} (${u.profile.email}) — Last login: ${lastLogin}`;
        });
        return { content: [{ type: 'text', text: [`Users inactive for ${days}+ days (${inactive.length} found):`, '─'.repeat(40), ...lines].join('\n') }] };
      }

      default:
        throw new Error(`Unknown tool: ${request.params.name}`);
    }

  } catch (err) {
    // Always return errors as readable text — never crash silently
    return {
      content: [{ type: 'text', text: `Error: ${(err as Error).message}` }],
      isError: true,
    };
  }
});

// ─── Start the server ──────────────────────────────────────────────────────────
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('personal-tools MCP server running (v3 — full suite)');
}

main().catch(console.error);
```

```bash
open .env
```

Add your Okta credentials:

```env
OKTA_ORG_URL=https://your-org.okta.com
OKTA_API_TOKEN=your_api_token_here
```

- `OKTA_ORG_URL` — your Okta org URL **without** `-admin`. Your admin console is `https://your-org-admin.okta.com` but the API URL must be `https://your-org.okta.com`. This is the most common mistake.
- `OKTA_API_TOKEN` — get it from your Okta admin console under **Security → API → Tokens**. Click **Create Token**, name it `claude-mcp`, copy the value immediately — Okta only shows it once.

## Troubleshooting

**Tools do not appear in Claude Desktop**

The most common cause is a wrong path in the config file. Run this and copy the exact output:

```bash
echo $(pwd)/dist/index.js
```

Also make sure you fully quit Claude Desktop — closing the window is not enough.

**"Cannot find module" error**

Make sure `"type": "module"` is in your `package.json`. This tells Node.js to use ES module imports.

**Weather returns wrong city or no data**

`wttr.in` handles most city names but can sometimes be confused by ambiguous names. Try being more specific: `Dublin, Ireland` instead of just `Dublin`.

**GitHub returns 403 Forbidden**

The GitHub public API has a rate limit of 60 requests per hour for unauthenticated requests. If you hit this, wait an hour or add a GitHub personal access token to your `.env`.

**RSS feed fails to load**

Some RSS feeds block automated requests. If a custom URL fails, try one of the built-in shortcuts (`bbc`, `hn`, `techcrunch`, `guardian`, `rte`) first to confirm the tool works, then try your custom URL again.

**Okta tools return a credential error**

Check your `.env` file. Make sure `OKTA_ORG_URL` does not include `-admin` in the URL. Confirm your API token is still valid in the Okta admin console under **Security → API → Tokens**.

**Check Claude Desktop logs**

If something is not working and you cannot tell why:

```bash
tail -50 ~/Library/Logs/Claude/mcp*.log
```

This shows the server startup output and any errors.

## Notes

- The Okta tools use basic list and filter endpoints and require a Read Only Admin token or equivalent
- `lookup_ip` uses ip-api.com which allows 45 requests per minute on the free tier
- `dns_lookup` uses Node.js native DNS resolution — no external API or rate limits apply
- `generate_password` uses Node.js `crypto.randomBytes` for cryptographically secure generation
- Never use `console.log()` in MCP server code — it writes to stdout and breaks the JSON-RPC connection. Always use `console.error()` for debug output
